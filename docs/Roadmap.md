# Roadmap.md

세계관/장르가 아직 미정이므로, Sprint 1의 목표를 원래 계획(세계관 확정)에서
**"엔진 검증"**으로 재설정할 것을 CTO가 제안한다. 세계관 결정은 CEO 재량이므로
Sprint 1 중 별도 Design Review에서 결정해도 파이프라인 개발과 병렬 진행 가능.

## Sprint 1 (2주) — 엔진 검증 + 세계관 확정

- LangGraph 기반 최소 파이프라인: Story Team → Editing Team → Publisher (3단계만)
- 더미/임시 세계관("테스트 월드")으로 챕터 1개 end-to-end 생성 성공시키기
- 병행: CEO가 실제 장르/세계관 방향 결정 (Design Review 1회 배정)
- 산출물: 동작하는 최소 파이프라인 + ADR-003 (세계관 확정 근거)

## Sprint 2 — 캐릭터 시스템 + Arc Team (신규)

- Character Team 추가, 캐릭터 시트 스키마 확정
- SQLite 캐릭터 DB 연동
- 확정된 세계관/장르 기준으로 실제 캐릭터 1~3명 설계
- **Arc Team 신설**: 세계관+캐릭터가 확정된 시점에 전체 단편의 장면 구조(에피소드 로드맵) 설계
  — "이 단편은 몇 장면인지, 각 장면의 목적/복선은 무엇인지"를 최초 1회 산출
- 산출물: 캐릭터 DB + 에피소드 로드맵 문서

## Sprint 3 — 연속성 메커니즘 + 토론 루프

- Emotion/Lore Team 추가
- **장면 간 상태 이어받기 구현**: 장면 N을 쓸 때 장면 N-1 요약 + 최신 캐릭터/세계관 DB를
  자동으로 불러오도록 파이프라인 연결 (Architecture.md 0단계 참고) — 이게 빠지면 장면마다
  안 이어지는 문제가 생기므로 이번 스프린트의 핵심 산출물
- Workflow.md의 비판 라운드 로직 구현 (종료 조건 포함)
- Arc 로드맵의 장면 2개를 실제로 "이어서" 완성해 연속성 검증

## Sprint 4 — Editor + Reader 피드백 루프

- Reader Team 추가 (가상 독자 피드백)
- 피드백이 다음 스프린트 우선순위에 실제로 반영되는지 검증 (Vision.md 성공 기준 충족 확인)

## 이후 (Level 3 예열)

- Prompt Review 결과를 자동으로 축적해 "어떤 프롬프트 변경이 품질을 올렸는가" 추적
- Agent 구조 자체의 버전 관리 (AgentSpec.md 변경 이력)
