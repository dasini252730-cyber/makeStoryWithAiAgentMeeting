# ADR-001: 오케스트레이션 프레임워크로 LangGraph를 선택

## 상태
승인됨 (2026-07-07, CEO 위임 → CTO 결정)

## 맥락
MUSE의 핵심 동작은 "초안 → 비판 → 수정 → 재토론"의 순환 구조이며,
Agent 간 판단 근거를 상태에 누적해 로그로 남겨야 한다 (No Magic 원칙).

## 검토한 대안
1. **LangGraph** — 상태그래프 기반, 조건부 루프/분기 제어에 강함
2. **직접 구현** — 유연하지만 초기 개발 비용이 크고, 검증된 패턴을 처음부터 재발명해야 함
3. **CrewAI/AutoGen** — 역할 기반 협업에 최적화되어 있으나, MUSE 고유의 "토론 종료 조건" 같은
   세밀한 제어 로직을 넣기엔 추상화 레벨이 높아 오히려 우회 구현이 필요할 수 있음

## 결정
LangGraph 채택.

## 근거
- 순환 구조(비판 라운드)를 1급 개념(그래프의 조건부 엣지)으로 표현 가능
- 상태(State)에 로그를 그대로 축적할 수 있어 로깅 원칙과 자연스럽게 맞물림
- Python/LangChain 생태계와 통합되어 있어 향후 도구 확장(웹검색, RAG 등) 용이

## 트레이드오프
- CrewAI 대비 초기 보일러플레이트 코드가 더 필요함 (역할 추상화를 직접 만들어야 함)
- 이 비용은 Sprint 1에서 최소 파이프라인(3단계)으로 먼저 검증하며 흡수

## 재검토 조건
Sprint 2 종료 시점에 "그래프 정의가 과도하게 복잡해지는가"를 기준으로 재검토.
