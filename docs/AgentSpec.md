# AgentSpec.md

각 Agent는 **직책 / 책임 / KPI / 입력 / 출력**을 가진다.

## Plot Liaison Agent (신규)

- **직책**: CEO ↔ 창작팀 사이의 플롯 논의 창구
- **책임**: CEO의 직관적 피드백("이거 상투적인데", "이 장면 좀 별로야")을 Arc/Story/Emotion
  Team이 실행 가능한 구체적 비판으로 번역하고, 반대로 팀들의 창작 결과와 트레이드오프를
  CEO가 판단하기 쉬운 형태(선택지 + 각각의 장단점)로 정리해 되돌려준다. Creative Director와의
  차이: Creative Director는 세계관 일관성만 승인/반려하지만, Plot Liaison은 "이야기가
  재밌는가, 뻔하지 않은가" 같은 취향/감의 영역을 다룬다.
- **KPI**: CEO 피드백 한 번에 팀이 정확히 알아듣는 비율 (재작업 없이 반영되는 정도)
- **입력**: CEO의 자유형 코멘트, 팀들의 초안/제안
- **출력**: 팀에게 줄 구체적 수정 지시 / CEO에게 줄 선택지 요약

## Creative Director Agent

- **직책**: 세계관·톤 일관성 총괄
- **책임**: 모든 팀 산출물이 세계관 설정과 충돌하지 않는지 최종 승인
- **KPI**: 세계관 모순 발견 건수(낮을수록 좋음), Lore Team과의 재작업 요청 횟수
- **입력**: 세계관 DB, 각 팀 초안
- **출력**: 승인/반려 + 반려 사유

## Project Manager Agent

- **직책**: 스프린트 진행 관리
- **책임**: 작업 분배, 팀 간 우선순위 조정, 비판 라운드 종료 판단
- **KPI**: 스프린트 목표 달성률, 평균 비판 라운드 수(수렴 속도)
- **입력**: Sprint 목표, 팀별 진행 상황
- **출력**: 일일 스탠드업 요약, 스프린트 리포트

## Arc Team (신규)

- **직책**: 전체 아크/에피소드 구조 설계
- **책임**: 확정된 세계관·캐릭터를 받아, 단편 전체를 몇 개 장면/챕터로 나눌지, 각 장면의 사건과
  목적, 복선을 어디서 심고 어디서 회수할지 설계. 다른 팀과 달리 **매 챕터마다 도는 게 아니라
  작업 최초 1회(또는 CEO의 방향 수정 시)만 실행**되며, 그 결과물(에피소드 로드맵)이 이후 모든
  Story Team 작업의 공통 입력값이 된다.
- **KPI**: 복선 회수율(심어놓은 복선 중 마무리된 비율), 에피소드별 분량 균형
- **입력**: 확정된 세계관, 캐릭터 시트, 장르/분량 방향
- **출력**: 에피소드 로드맵 (장면별 목적, 등장 캐릭터, 복선 태그)

## Story Team

- **직책**: 플롯 초안 생성
- **책임**: 챕터 단위 플롯 여러 안 제시
- **KPI**: Creative Director 1차 승인률
- **입력**: Arc Team의 에피소드 로드맵(이번 장면의 목적/복선 태그), 직전 장면 요약, 세계관 설정
- **출력**: 플롯 초안 N안 + 각 안의 장단점

## Character Team

- **직책**: 캐릭터 정합성 관리
- **책임**: 대사/행동이 캐릭터 시트와 일치하는지 검증, 캐릭터 성장 곡선 관리
- **KPI**: 캐릭터 일관성 위반 발견 건수 (독자 피드백 기준 역산 가능)
- **입력**: 캐릭터 시트, 초안
- **출력**: 수정 제안 + 근거

## Emotion Team

- **직책**: 감정선/페이싱 관리
- **책임**: 챕터 내 긴장-이완 곡선이 장르 관습에 맞는지 검토
- **KPI**: Reader Team이 측정한 몰입도 시뮬레이션 점수
- **입력**: 초안, 장르별 페이싱 가이드
- **출력**: 페이싱 조정 제안

## Editing Team

- **직책**: 문장/구조 교정
- **책임**: 문법, 문체 일관성, 가독성
- **KPI**: 교정 필요 문장 비율 (챕터 진행할수록 감소해야 함)
- **입력**: 초안
- **출력**: 교정본 + diff

## Reader Team

- **직책**: 가상 독자 피드백 시뮬레이션
- **책임**: "왜 이 지점에서 이탈할 것 같은지" 등 예측 피드백 생성
- **KPI**: (장기) 실제 독자 피드백과의 예측 정합도
- **입력**: 최종 초안
- **출력**: 피드백 리포트 (다음 스프린트 우선순위 입력값)

## Lore Team

- **직책**: 세계관 DB 관리
- **책임**: 설정 저장, 모순 자동 탐지 (Chroma 의미 검색 활용)
- **KPI**: 모순 조기 발견률
- **입력**: 확정된 세계관 설정, 신규 초안
- **출력**: 모순 경고, 세계관 DB 업데이트

## Publisher Agent

- **직책**: 최종 조립/출력
- **책임**: 모든 팀 산출물을 최종본으로 병합, 포맷팅
- **KPI**: 재작업 없이 발행되는 비율
- **입력**: 승인된 최종 초안
- **출력**: 발행용 완성 챕터
