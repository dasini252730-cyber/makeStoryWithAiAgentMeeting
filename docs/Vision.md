# Vision.md

## 한 줄 정의

AI가 스스로 기획하고, 토론하고, 비판하고, 수정하며 성장하는 **창작 조직**을 만든다.

최종 산출물은 "웹소설 생성기"가 아니라, **AI 창작 회사를 운영하는 운영체제(Creative OS)**다.

## 왜 이렇게 접근하는가

일반적인 "AI로 소설 쓰기" 프로젝트는 하나의 거대 프롬프트나 파이프라인으로 수렴한다.
그 결과 품질 상한이 "프롬프트 하나가 얼마나 좋은가"에 묶인다.

MUSE는 반대로 간다: 여러 역할(Agent)이 서로 다른 관점에서 초안을 만들고, 비판하고,
합의에 도달하는 **과정**을 시스템화한다. 품질 상한이 "조직의 협업 구조"에 묶이도록 만드는 것.

## 성장 단계

| Level | 목표 | 상태 |
|---|---|---|
| 1 | AI가 소설을 쓴다 | ❌ (목표 아님) |
| 2 | AI가 서로 토론하며 소설을 만든다 | ⭕ 현재 목표 |
| 3 | AI가 스스로 성장한다 (프롬프트/워크플로우/에이전트 자체 개선) | ⭐ 장기 목표 |

## 개발 원칙

1. **Human First** — 사람은 절대 글을 쓰지 않는다. 방향만 제시한다.
2. **Every AI has a role** — 모든 Agent는 직책, 책임, KPI를 가진다 (`AgentSpec.md`).
3. **No Magic** — 모든 판단에는 근거 로그가 남는다. "왜 이렇게 판단했는가"가 항상 추적 가능해야 한다.
4. **Everything Evolves** — Prompt, Workflow, Agent 구조 모두 버전 관리되고 개선된다.

## 성공 기준 (Level 2 기준)

- 사람이 세계관/캐릭터 승인만 했을 뿐인데, 완결된 챕터가 나온다.
- 각 챕터마다 "이 결정을 왜 내렸는가"에 대한 Agent 간 토론 로그가 존재한다.
- 독자 피드백(또는 Reader Agent 시뮬레이션)이 다음 스프린트 우선순위에 실제로 반영된다.
